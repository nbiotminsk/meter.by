# Requirements Document

## Introduction

Улучшение секции "Пакетные предложения" на странице цен (prices.html) для обеспечения единообразного размера карточек, выравнивания кнопок и интеграции с калькулятором стоимости. Карточки должны иметь современный дизайн с иконками и функциональными кнопками добавления в корзину.

## Requirements

### Requirement 1

**User Story:** Как пользователь страницы цен, я хочу видеть все карточки пакетных предложений одинакового размера, чтобы интерфейс выглядел аккуратно и профессионально.

#### Acceptance Criteria

1. WHEN пользователь просматривает секцию пакетных предложений THEN система SHALL отобразить все карточки одинаковой высоты
2. WHEN карточки отображаются THEN система SHALL использовать CSS Flexbox или Bootstrap классы для выравнивания высоты
3. WHEN контент карточек различается по длине THEN система SHALL обеспечить равномерное распределение пространства

### Requirement 2

**User Story:** Как пользователь, я хочу видеть все кнопки "Выбрать пакет" на одном уровне, чтобы интерфейс был удобным для взаимодействия.

#### Acceptance Criteria

1. WHEN карточки пакетов отображаются THEN система SHALL выровнять все кнопки "Выбрать пакет" по нижнему краю карточек
2. WHEN используется Bootstrap card layout THEN система SHALL применить класс h-100 для равной высоты
3. WHEN кнопки размещаются THEN система SHALL использовать margin-top: auto для выравнивания по низу

### Requirement 3

**User Story:** Как пользователь, я хочу видеть соответствующие иконки для каждого пакета, чтобы быстро понимать тип предложения.

#### Acceptance Criteria

1. WHEN отображается "Базовый пакет (Тепло)" THEN система SHALL показать иконку bi-thermometer-half
2. WHEN отображается "Премиум пакет (Тепло)" THEN система SHALL показать иконку bi-star-fill
3. WHEN отображается "Комплект для воды" THEN система SHALL показать иконку bi-droplet-fill
4. WHEN отображается "Умный дом" THEN система SHALL показать иконку bi-house-gear-fill
5. WHEN отображается "Бизнес решение" THEN система SHALL показать иконку bi-building
6. WHEN отображается "Годовое обслуживание" THEN система SHALL показать иконку bi-calendar-check

### Requirement 4

**User Story:** Как пользователь, я хочу добавлять пакеты в корзину калькулятора одним кликом, чтобы быстро рассчитать общую стоимость.

#### Acceptance Criteria

1. WHEN пользователь кликает кнопку "Выбрать пакет" THEN система SHALL добавить пакет в корзину калькулятора стоимости
2. WHEN пакет добавлен в корзину THEN система SHALL обновить общую сумму в калькуляторе
3. WHEN пакет добавлен THEN система SHALL изменить текст кнопки на "В корзине" с иконкой bi-check-circle-fill
4. WHEN пакет добавлен THEN система SHALL изменить цвет кнопки на btn-success
5. WHEN пакет добавлен THEN система SHALL отключить кнопку (disabled) для предотвращения повторного добавления

### Requirement 5

**User Story:** Как пользователь, я хочу видеть визуальную обратную связь при добавлении пакета в корзину, чтобы понимать, что действие выполнено успешно.

#### Acceptance Criteria

1. WHEN пользователь добавляет пакет в корзину THEN система SHALL показать уведомление SweetAlert2 об успешном добавлении
2. WHEN уведомление отображается THEN система SHALL показать название добавленного пакета и его стоимость
3. WHEN пакет добавлен THEN система SHALL автоматически прокрутить страницу к секции калькулятора
4. WHEN пользователь пытается добавить уже выбранный пакет THEN система SHALL показать информационное сообщение о том, что пакет уже в корзине

### Requirement 6

**User Story:** Как пользователь, я хочу иметь возможность удалить пакет из корзины, чтобы изменить свой выбор.

#### Acceptance Criteria

1. WHEN пользователь удаляет пакет из корзины калькулятора THEN система SHALL вернуть кнопку пакета в исходное состояние
2. WHEN кнопка возвращается в исходное состояние THEN система SHALL изменить текст на "Выбрать пакет" с иконкой bi-box-seam
3. WHEN кнопка сбрасывается THEN система SHALL изменить цвет обратно на btn-primary
4. WHEN кнопка сбрасывается THEN система SHALL включить кнопку (enabled) для повторного выбора

### Requirement 7

**User Story:** Как пользователь, я хочу видеть четкую структуру информации в каждой карточке пакета, чтобы легко сравнивать предложения.

#### Acceptance Criteria

1. WHEN карточка пакета отображается THEN система SHALL показать заголовок с иконкой в header карточки
2. WHEN отображается ценовая информация THEN система SHALL показать оригинальную цену зачеркнутой, цену со скидкой крупным шрифтом, и размер экономии в badge
3. WHEN отображается список услуг THEN система SHALL показать каждую услугу с иконкой bi-check-circle и зеленым цветом
4. WHEN карточка содержит много услуг THEN система SHALL ограничить высоту списка и добавить прокрутку при необходимости

### Requirement 8

**User Story:** Как пользователь, я хочу видеть популярные пакеты выделенными, чтобы быстро найти рекомендуемые предложения.

#### Acceptance Criteria

1. WHEN отображается популярный пакет THEN система SHALL добавить badge "Популярный" с иконкой bi-star-fill
2. WHEN популярный пакет отображается THEN система SHALL использовать border-warning для выделения
3. WHEN популярный пакет показывается THEN система SHALL применить легкое увеличение масштаба (transform: scale(1.02))
4. WHEN популярный пакет отображается THEN система SHALL добавить дополнительную тень для визуального выделения

### Requirement 9

**User Story:** Как пользователь калькулятора, я хочу указывать количество для каждого пакета или услуги, чтобы рассчитать стоимость для нескольких единиц.

#### Acceptance Criteria

1. WHEN пакет добавлен в корзину THEN система SHALL отобразить поле для ввода количества рядом с названием пакета
2. WHEN пользователь изменяет количество THEN система SHALL автоматически пересчитать общую стоимость
3. WHEN количество равно 1 THEN система SHALL показать количество по умолчанию без дополнительного ввода
4. WHEN пользователь вводит количество больше 1 THEN система SHALL отобразить "x[количество]" рядом с названием пакета
5. WHEN количество изменяется THEN система SHALL обновить итоговую сумму в реальном времени
6. WHEN пользователь вводит некорректное количество (0 или отрицательное) THEN система SHALL установить количество равным 1
7. WHEN пакет удаляется из корзины THEN система SHALL сбросить количество и пересчитать общую сумму
